# Нефункциональные требования: атрибуты качества ERP-системы

Фрагмент нефункциональных требований для ERP-системы ремонтно-строительной компании. Требования сгруппированы по удобству использования, производительности, безопасности, защите данных, доступности и надежности.

## 6.1. Удобство использования

| ID | Требование |
| --- | --- |
| УИ-2 | Функция создания отчета должна быть доступна из личного кабинета мастера не более чем за один переход (клик) от главной страницы. |
| УИ-3 | Интерфейс заполнения отчета должен содержать валидацию полей, исключающую отправку некорректных данных. |

## 6.2. Производительность

| ID | Требование |
| --- | --- |
| Про-1 | Система должна поддерживать одновременную работу не менее 50 пользователей (менеджеры и мастера) без деградации основных операций: создание отчетов, сохранение работ, формирование документов. |
| Про-2 | Генерация Excel-отчетов по объектам с количеством позиций до 500 не должна занимать более 60 секунд. |
| Про-3 | Время отклика API при сохранении строки работы в отчете не должно превышать 3 секунд при скорости соединения от 10 Мбит/сек. |

## 6.3. Безопасность

| ID | Требование |
| --- | --- |
| Без-1 | Доступ к финансовым итогам проекта и закупочным ценам должен быть ограничен ролями "Менеджер проекта" и "Администратор". Пользователь с ролью "Мастер" не должен иметь доступа к маржинальности. |
| Без-3 | Передача данных между клиентом и сервером должна осуществляться по защищенному протоколу HTTPS. |
| Без-4 | Система должна реализовывать ролевую модель доступа (RBAC), ограничивающую доступ к операциям и данным в зависимости от роли пользователя. |

## 6.4. Защита

| ID | Требование |
| --- | --- |
| Защ-1 | Система должна выполнять автоматическое резервное копирование базы данных не реже 1 раза в 24 часа. |
| Защ-2 | Резервные копии должны храниться не менее 30 календарных дней. |

## 6.5. Доступность

| ID | Требование |
| --- | --- |
| Дос-1 | Веб-приложение должно быть доступно в режиме 24/7 с коэффициентом доступности 99.5%, так как закупки и отчеты могут вноситься вне стандартного рабочего графика. |

## 6.6. Надежность

| ID | Требование |
| --- | --- |
| Над-1 | Механизм черновиков: если связь разрывается во время заполнения отчета, система должна сохранять введенные данные, чтобы пользователь мог продолжить с того же места. |
