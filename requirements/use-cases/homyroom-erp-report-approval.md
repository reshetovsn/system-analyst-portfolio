# ВИ-М1: Контроль за выполнением работ и утверждение отчетности

Фрагмент спецификации требований для ERP-системы ремонтно-строительной компании. Use Case описывает проверку отчетов исполнителей, фиксацию цен, принятие работ и обновление план-факта по проекту.

| Поле | Описание |
| --- | --- |
| Уникальный код и название | ВИ-М1: Контроль за выполнением работ и утверждение отчетности |
| Контекст использования | Менеджер проверяет фактические объемы работ/закупок, проставляет цены и формирует закрывающие документы. |
| Область действия | Веб-приложение |
| Основное действующее лицо | Менеджер |
| Предусловие | Менеджер авторизован. В системе есть активный объект. Исполнители отправили хотя бы один отчет на проверку. |
| Гарантии успеха | Отчет принят, цены зафиксированы, данные вошли в общую статистику проекта.<br>Отчет отклонен, отправлен на доработку. |
| Триггер | Поступление уведомления о новом отчете. |
| Базовый сценарий | Успешная приемка отчета:<br>1. Менеджер выбирает нужный объект и открывает раздел "Отчеты".<br>2. Система отображает список отчетов со статусом "На проверке".<br>3. Менеджер выбирает конкретный отчет для просмотра.<br>4. Система выводит детализацию: названия работ, объемы, исполнителей и поля для ввода цен.<br>5. Менеджер корректирует или проставляет себестоимость и цену для заказчика.<br>6. Менеджер выставляет такелажные работы в процентах.<br>7. Менеджер выбирает действие "Принять отчет".<br>8. Система меняет статус отчета на "Принят".<br>9. Система блокирует отчет для редактирования.<br>10. Система обновляет план-факт по проекту. |
| Расширения | Отчет содержит ошибки или некорректные данные:<br>7.a. Менеджер выбирает действие "Отправить на доработку".<br>7.b. Система меняет статус отчета на "На доработке".<br>7.c. Система уведомляет исполнителя о возврате отчета на доработку.<br>7.d. Система открывает исполнителю доступ к редактированию.<br><br>Есть работа, отсутствующая в смете:<br>10.a. Система обновляет раздел "Доп. работы" и добавляет новые работы, внесенные исполнителем.<br>10.b. Система автоматически рассчитывает процент всех доп. работ относительно бюджета.<br><br>Фактический объем работы превышает объем, указанный в первоначальной смете:<br>10.c. Система обновляет раздел "Перерасход". Рассчитывает объем перерасхода и цену.<br>10.d. Система автоматически рассчитывает процент перерасхода относительно бюджета.<br>10.e. Система подсвечивает эти работы в разделе "Ход работы". |
