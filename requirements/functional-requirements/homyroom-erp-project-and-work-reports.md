# Функциональные требования: управление проектом и отчеты по работам

Фрагмент спецификации требований для ERP-системы ремонтно-строительной компании. Раздел описывает создание объекта, базовые настройки проекта и формирование отчетов по выполненным работам.

## Проект.Управление: создание объекта и базовые настройки

| Функция | Требование |
| --- | --- |
| Создать | Система должна предоставлять менеджеру возможность создания нового объекта с указанием названия и базовой информации. |
| Загрузить смету | Система должна обеспечивать импорт сметы работ и спецификации материалов из внешних файлов формата XLSX. |
| Назначить мастера | Система должна позволять менеджеру закреплять ответственных исполнителей (мастеров) за конкретными разделами сметы и/или за конкретными видами работ. |
| Активировать | Система должна переводить проект в статус "В работе", открывая исполнителям доступ к формированию отчетов. |
| Статистика | Система должна автоматически обновлять финансовые показатели проекта при изменении статусов отчетов. |

## Отчет.Работы: формирование отчетов по выполненным работам

| Функция | Требование |
| --- | --- |
| Выбрать объем | Система должна позволять мастеру вводить фактический объем выполненных работ на основе плановой сметы. |
| Расчет остатка | Система должна в реальном времени вычислять остаток по каждой позиции сметы на основе введенных данных. |
| Дополнительная работа | Система должна позволять мастеру добавлять работы, отсутствующие в смете, выбирая их из справочника или создавая новые позиции. |
| Ограничение цен | Система должна отображать мастеру только его цены (себестоимость), скрывая цену для заказчика. |
| Такелаж | Система должна предоставлять возможность включения такелажных работ с автоматическим применением процента наценки (по умолчанию 0,8%). |
| Заблокировать | Система должна блокировать отчет для редактирования исполнителем сразу после отправки на проверку. |
