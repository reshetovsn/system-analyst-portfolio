# REST API: ресурс "Проекты"

Фрагмент описания REST API для ERP-системы ремонтно-строительной компании. Раздел включает общие правила взаимодействия с API, реестр методов ресурса "Проекты" и детальное описание двух методов.

## 5.2. Программные интерфейсы

## 5.2.0 Общие правила взаимодействия с API

| Параметр | Значение |
| --- | --- |
| Базовый URL | `/api/v1/` |
| Протокол | HTTPS |
| Формат данных | JSON |
| Кодировка | UTF-8 |
| Аутентификация | Session-based авторизация |
| Валидация | Права доступа проверяются на стороне сервера: администратор, менеджер, мастер. |

## Стандартные коды ответов

| Код | Описание |
| --- | --- |
| `200 OK` | Успешное выполнение запроса: получение данных или обновление объекта. |
| `201 Created` | Объект успешно создан. |
| `204 No Content` | Объект успешно удален. |
| `400 Bad Request` | Ошибка в параметрах запроса или нарушение бизнес-логики. |
| `401 Unauthorized` | Пользователь не авторизован. |
| `403 Forbidden` | Недостаточно прав. |
| `404 Not Found` | Запрашиваемый ресурс не найден. |
| `500 Internal Server Error` | Ошибка на стороне сервера. |

## Обработка числовых данных

Числовые значения, например количество и цена, передаются в API как `number`.

## 5.2.1 Ресурс "Проекты" (Projects)

## 5.2.1.1 Краткий реестр ендпоинтов

| Метод | Путь | Описание |
| --- | --- | --- |
| GET | `/api/v1/projects/` | Получение информации по всем объектам. |
| POST | `/api/v1/projects/` | Создание нового объекта. |
| GET | `/api/v1/projects/{id}/` | Получение информации по конкретному объекту. |
| PUT | `/api/v1/projects/{id}/` | Редактирование объекта. |
| DELETE | `/api/v1/projects/{id}/` | Удаление объекта. |
| GET | `/api/v1/projects/{id}/estimates/` | Получение списка всех смет, связанных с конкретным объектом. |
| GET | `/api/v1/projects/{id}/totals/` | Финансовая сводка проекта. |
| GET | `/api/v1/projects/{id}/overspending/` | Перерасход. |
| GET | `/api/v1/projects/{id}/additional/` | Дополнительные работы. |

## 5.2.1.2 Детальное описание методов

## 1. Получение списка проектов

| Параметр | Значение |
| --- | --- |
| Метод | GET |
| URL | `/api/v1/projects/` |
| Описание | Получение реестра всех проектов с краткой информацией о связанных сметах и справочниках. |
| Ответ | `200 OK` |

Ответ содержит массивы `estimates` и `dictionaries`, описанные в моделях `EstimateShortDTO` и `DictionaryShortDTO`.

```json
{
  "id": 1,
  "name": "string",
  "estimates": [
    {
      "id": 1,
      "name": "string",
      "status": "draft",
      "full_price_project": 100000
    }
  ],
  "dictionaries": [
    {
      "id": 1,
      "name": "string",
      "work_type": "work"
    }
  ]
}
```

## 2. Создание нового проекта

| Параметр | Значение |
| --- | --- |
| Метод | POST |
| URL | `/api/v1/projects/` |
| Описание | Инициализация нового строительного объекта. |
| Тело запроса | JSON |
| Ответ | `201 Created` |

```json
{
  "name": "string"
}
```

```json
{
  "id": 1,
  "name": "string",
  "open": false,
  "estimates": [],
  "dictionaries": []
}
```
